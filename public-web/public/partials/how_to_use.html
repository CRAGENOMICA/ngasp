Depending on your initial datasets you can use a different pipeline:


  • BAM diploid individuals


    If you have individual diploid BAM files, the pipeline 1 is suitable for your analysis. You will need the reference sequence, and optional outgroup sequence(s) and/or annotation file (GTF v2).


    Right now, the pipeline works independently for each chromosome, so you will have to run each chromosome separately (one run of this pipeline per chromosome).


    You can optionally analyze a number of different regions, chose the window size, chose the regions of the genome to study, choose the functional regions (synonymous, non-synonymous, silent, coding, etc…) you want to analyze. If you want to analyze many different options (e.g., synonymous and non-synonymous, different size of the sliding windows…), it is posible to keep intermediate results (that is, tfasta files) to run again a subset of the pipeline.


  • Other datasets:


    gVCF files


    BAM files from pooled data


We are working in opening a number of different pipelines in order to work with gVCF and also with pooled data. Right now it is possible to analyze this kind of datasets using available related software.
